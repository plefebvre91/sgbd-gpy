-> Avancees

l'ensemble des jeux critiqués disponibles sur une plateforme donnée, classés par catégorie ;

Ensemble des jeux:
SELECT *
FROM jeu;

Ensemble des jeux classés par idCategorie:
SELECT jeu.* , appartient.idCategorie
FROM jeu
INNER JOIN appartient ON jeu.idJeu = appartient.idJeu
ORDER BY idCategorie ASC;

Ensemble des jeux classés par catégorie:
SELECT ssrqt.idJeu, ssrqt.nomJeu, categorie.nomCategorie
FROM (
     SELECT jeu.* , appartient.idCategorie
     FROM jeu
     INNER JOIN appartient ON jeu.idJeu = appartient.idJeu
     ) as ssrqt
     LEFT OUTER JOIN categorie
ON ssrqt.idCategorie  = categorie.idCategorie
ORDER BY categorie.nomCategorie ASC;

Requête complète qui ne fonctionne pas :
SELECT * 
FROM (((jeu INNER JOIN appartient 
ON jeu.idJeu = appartient.idJeu) INNER JOIN categorie 
ON appartient.idCategorie = categorie.idCategorie) INNER estDisponible
ON jeu.idJeu = estDisponible.idJeu) INNER JOIN plateforme
ON estDisponible.idPlateforme = plateforme.idPlateforme
WHERE nomPlateforme = "ps3"
GROUP BY categorie
ORDER BY categorie ASC;

pour un joueur donné, la liste des commentaires se référant à un jeu dans sa catégorie préférée,
disponible sur sa plateforme préférée ;
pour un commentaire, la liste des joueurs qui l'ont apprécié.


-> Statistiques

les joueurs, classés selon le nombre de jeux qu'ils ont notés ;
la liste des n commentaires les plus récents ;
le commentaire qui laisse le moins indifférent (celui qui a reçu le plus de jugements) ;

chaque commentaire à un indice de confiance égal à 1+c / 1+d
où c est le nombre d'appréciations positives et d le nombre d'appréciations négatives ; classer
les commentaires selon leur indice de confiance.

Ajout, suppression, modification d'un jeu, d'un joueur, d'un commentaire. On prendra soin à
ce que les requêtes de mise à jour conservent l'intégrité et la cohérence de la base.
Toute autre requête que vous jugeriez utile pour la gestion de cette base.
